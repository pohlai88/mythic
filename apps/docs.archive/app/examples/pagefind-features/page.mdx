# Pagefind Features Demonstration

This page demonstrates the advanced search capabilities of **Pagefind** in Nextra 4.

## 1. Indexing Remote MDX Content

Pagefind can index content from async components:

```tsx
export async function Stars() {
  const response = await fetch('https://api.github.com/repos/shuding/nextra')
  const repo = await response.json()
  const stars = repo.stargazers_count
  return <b>{stars}</b>
}
```

**Try searching for**: "stars" or "GitHub" - this content will be indexed!

## 2. Indexing Dynamic Content

Pagefind indexes dynamic content written in Markdown/MDX:

**Current Year**: {new Date().getFullYear()}

**Try searching for**: The current year - it will be indexed!

## 3. Indexing Imported Components

Pagefind can index content from imported files. Check the source code to see how components are imported and their content indexed.

## 4. Code Block Search

Code blocks can be included in search (if enabled in `next.config.mjs`):

```javascript
// This code block can be searched if codeblocks: true
function example() {
  return 'searchable code'
}
```

## How to Test

1. **Open the search** (click search icon in navbar)
2. **Search for**: "stars", "GitHub", current year, or any content on this page
3. **Verify**: Results should include this page and relevant content

## Configuration

Search is configured in `next.config.mjs`:

```javascript
search: {
  codeblocks: false, // Set to true to enable code block search
}
```

## Benefits

- ✅ **Faster**: Rust-powered search engine
- ✅ **Better Results**: Superior relevance ranking
- ✅ **More Content**: Indexes remote/dynamic content
- ✅ **Automatic**: Works out of the box for MDX pages
